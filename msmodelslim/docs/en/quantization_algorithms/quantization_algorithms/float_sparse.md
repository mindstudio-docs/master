# Floating-Point Sparsity: ADMM-based Model Sparsification Algorithm

## Overview

- **Problem**: Although existing W8A8S quantization methods support sparse weight quantization, the sparsity rate remains low, making it difficult to achieve an ideal compression effect on the compression unit of the Atlas 300I Duo inference card. Furthermore, meeting accuracy requirements often necessitates quantization fallback for specific network layers, which significantly reduces the inference performance of the model. Therefore, we propose to sparsify the floating-point weights to achieve a higher compression rate by using the hardware compression unit, thereby significantly improving inference performance while ensuring model accuracy.
- **Objective**: Implement floating-point sparsity for the model by using the alternating direction method of multipliers (ADMM) algorithm, and maintain accuracy at critical positions by using L2 quantization to achieve a high compression rate while ensuring model performance.

## Principle and Implementation

### Principle

**Core Concepts**

1. **ADMM optimization**: The algorithm solves constrained optimization problems by using the ADMM algorithm to identify the optimal weight sparsity pattern.
2. **Activation statistics**: The algorithm collects activation statistics by using forward hooks to construct the Hessian matrix.
3. **Iterative sparsity**: The algorithm balances the sparsity rate and model accuracy by using multiple iterations to optimize the sparsity pattern.
4. **Accuracy protection**: The algorithm maintains accuracy at critical positions by using L2 quantization to prevent the excessive compression of key weights.

**Algorithm Process**

1. **Preprocessing**: Install the forward hooks, collect activation statistics, and construct the Hessian matrix.
2. **ADMM sparsification**: Identify the optimal sparsity pattern by using the ADMM algorithm.
3. **Iterative optimization**: Refine the sparsity results by using multiple iterations.
4. **Accuracy protection**: Identify critical weight positions and apply L2 quantization to maintain accuracy.
5. **Module deployment**: Convert the sparsified modules into quantized modules.

#### Core Algorithm Steps

1. **Statistics collection**:
   - Install forward hooks to collect input activation data.
   - Accumulate the Hessian matrix: `H += X^T * X`.
   - Calculate the row scaling factor: `scaler_row += ||X_i||_2^2 / n_samples`.

2. **ADMM-based sparsification**:
   - Normalize the Hessian matrix and weights.
   - Set the initial penalty parameter: `rho0 = PERCDAMP * mean(diag(H))`.
   - Compute the inverse of the Hessian matrix.
   - Execute the ADMM main loop:
     - Project to the sparse space: `sparse_weights = (weights + lambda) * mask`.
     - Update the Lagrange multiplier: `lambda += (weights - sparse_weights)`.
     - Update the weights: `weights = H_inv * (H*weights + rho*(sparse_weights - lambda))`.

3. **Accuracy protection**:
   - Determine the importance measure by using the product of the quantization error and the scaling factor.
   - Select the top-k% most important positions to maintain precision.
   - Identify critical weight positions and apply L2 quantization to maintain accuracy.

## Application Requirements

- **High compression requirements**: This algorithm is suitable for model deployment scenarios that require a high compression ratio.
- **Accuracy-sensitive**: The accuracy protection mechanism maintains the precision of critical weights during the compression process.
- **Computational cost**: The ADMM algorithm requires multiple iterations, which leads to high computational cost and slower processing speed.
- **Memory requirements**: This algorithm stores the Hessian matrix and activation statistics and therefore consumes large amounts of GPU memory.
- **Usage limitations**:
  - The weights generated by using the current algorithm must undergo further compression by using the hardware compression unit of the Atlas 300I Duo inference card. This is necessary to effectively reduce the model size, decrease GPU memory usage during deployment, and improve performance.
  - The Atlas 300I Duo inference card does not support the `bfloat` data type. Therefore, when performing floating-point sparsity on a model, the `torch_dtype` field in the `config.json` file within the model path must be manually changed to `float16`.
  - Only `layer-wise quantization` within the `v1 framework` is supported.
  - Currently, floating-point sparsity is supported only for the `nn.Linear` module.
  - Activation statistics must be collected by using a calibration dataset. The number of token IDs within the calibration data must be greater than or equal to 2048.
  - You are advised to gradually adjust the sparsity ratio to a value close to `0.3`.

## Performance

### Benefits

1. **High compression ratio**: The algorithm achieves a high sparsity ratio and significant compression effects by using the ADMM algorithm.
2. **Accuracy protection**: The algorithm intelligently identifies critical weight positions to avoid the loss of key information.
3. **Adaptive optimization**: The sparsity policy is automatically adjusted by using activation statistics.
4. **Layer-wise quantization**: Layer-wise quantization is supported to reduce memory usage.

### Limitations

1. **Computational overhead**: The time required for model sparsification increases due to ADMM iterations and Hessian matrix computation.
2. **GPU memory usage**: Additional statistics and intermediate results must be stored, which increases GPU memory consumption.
3. **Parameter tuning**: Parameters such as the sparsity ratio must be adjusted based on the specific model.

## FAQ

### Incompatibility between Floating-Point Sparsity and W8A8 Sparse Weight Quantization

**Symptom**: A user attempts to further sparsify a model by using the floating-point sparsity algorithm after the model has already undergone INT8 sparse weight quantization (W8A8S).

**Solution**: The floating-point sparsity algorithm (W16A16S) and W8A8S represent two different technical approaches and cannot be used together. Repeated sparsification accumulates accuracy loss, which may severely affect the inference performance of the model.

### Excessively High Sparsity Ratio

**Symptom**: The model accuracy decreases significantly due to an excessively high sparsity ratio.

**Solution**: Decrease the value of the `sparse_ratio` parameter. You are advised to gradually adjust the value to approximately 0.3.

### Matrix Inversion Failure Caused by Insufficient Calibration Data Length

**Symptom**: An error occurs, indicating that the matrix inversion failed while processing a large model.

**Solution**: Increase the length of each data record within the calibration dataset to ensure the number of token IDs after tokenization is greater than or equal to 2048.

### GPU memory overflow caused by an excessive number of calibration samples

**Symptom**: A GPU memory overflow error occurs during the processing of a large model.

**Solution**: Reduce the number of samples within the calibration dataset, or perform floating-point sparsification by using a machine with larger GPU memory.
