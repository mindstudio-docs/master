# ReStepAdaptor

## Function

A sampling optimization adapter class used to search and optimize the sampling steps of a stable diffusion model to improve inference efficiency.

## Prototype

```python
ReStepAdaptor(pipeline, config)
```

## Parameter Description

| Parameter | Input/Return | Description | Constraints |
| ------ | ------ | ------ | ------ |
| pipeline | Input | An OpenSoraPipeline instance. | Required.<br>Data type: OpenSoraPipeline. |
| config | Input | Sampling optimization search configuration. | Required.<br>Data Type: ReStepSearchConfig.<br>Contains configuration parameters such as videos_path and save_dir. |

## Main Methods

### search()

Executes the sampling step search process to optimize the sampling steps of the stable diffusion model, thereby improving inference efficiency. This method analyzes the model's behavior at different timesteps to identify the key timesteps that have the greatest impact on generation quality, thus reducing computational cost while maintaining generation quality.

**Returns:**

- Data type: list[float]

- Meaning: The optimized list of timesteps obtained from the search. Each element represents a sampling time point within the range [0,1].

## Sample

```python
from msmodelslim.pytorch.multi_modal.sampling_optimization import ReStepSearchConfig, ReStepAdaptor

# Set the search configuration.
config = ReStepSearchConfig(
    videos_path='/path/of/your/calibration_videos',  # Calibration video directory generated by the original model.
    save_dir='/path/to/save/searched_results',       # Directory to save search results.
    neighbour_type='uniform',                        # Neighborhood type used during sampling.
    monte_carlo_iters=5                              # Number of video samples during sampling.
)

# Create an adapter and perform the search.
restep_adaptor = ReStepAdaptor(pipeline, config)
scheduler_timestep: list = restep_adaptor.search()
```

## Usage Instructions

1. First, you need to generate a set of calibration videos using the original number of sampling steps for subsequent search optimization.

2. Configure ReStepSearchConfig, specifying the calibration video path and other search parameters.

3. Create a ReStepAdaptor instance and call the search() method to perform the search.

4. After the search is complete, the returned optimized timesteps can be used for inference, typically achieving better inference efficiency.

## Notes

1. The quality of calibration videos affects the accuracy of search results. It is recommended that you use representative video samples.

2. The search process may take some time, depending on monte_carlo_iters and the number of videos.

3. When applying the optimized timesteps to inference, ensure that the model and data processing pipeline remain consistent with those used during the search.
