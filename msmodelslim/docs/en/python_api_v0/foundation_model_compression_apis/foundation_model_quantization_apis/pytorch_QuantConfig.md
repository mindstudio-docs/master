# QuantConfig

## Function

Quantization parameter configuration class, which stores the parameters configured during the quantization process.

## Prototype

```python
QuantConfig(a_bit=8, w_bit=8, disable_names=None, dev_type='cpu', dev_id=None, act_method=1, pr=1.0, w_sym=True, mm_tensor=True, w_method='MinMax', co_sparse=False, fraction=0.01, nonuniform=False, is_lowbit =False, do_smooth=False, use_sigma=False, sigma_factor=3, disable_last_linear: bool=True, use_kvcache_quant=False, is_dynamic=False, open_outlier=True, group_size=64, percdamp=0.01, pdmix=False)
```

## Parameters

| Parameter | Input/Return | Description | Constraints |
| ------ | ------ | ------ | ------ |
| a_bit | Input | Activation quantization bit. | Optional.<br>Data type: int.<br>Optional values are 4, 8, and 16. The default value is 8.<br>In large model quantization scenarios, it can be configured as 4, 8, or 16. In per_group scenarios, it must be configured as 8 or 16 (a_bit=8 is currently only used in w4a8 quantization). When the is_dynamic parameter is configured as True, using per-token dynamic quantization, it must be configured as 4 or 8.<br>In large model sparse quantization scenarios, it must be configured as 8.<br>The w4a4 scenario only supports per-token dynamic quantization, only supports configuring is_dynamic as True, and other parameters are not applicable. This scenario currently only supports the Qwen3 series dense models, and the use of the outlier suppression feature is not recommended. |
| w_bit | Input | Weight quantization bit. | Optional.<br>Data type: int.<br>Optional values are 4 and 8. The default value is 8.<br>In large model quantization scenarios, it can be configured as 4 or 8. When the is_dynamic parameter is configured as True, using per-token dynamic quantization, it must be configured as 4 or 8.<br>In large model sparse quantization scenarios, it must be configured as 4. |
| disable_names | Input | Names of nodes whose weights need to be excluded from quantization, i.e., the names of quantization layers for manual fallback.<br>If the accuracy is too poor, it is recommended to fallback quantization-sensitive layers, such as classification layers, input layers, and detection head layers. | Optional.<br>Data type: object. |
| dev_type | Input | Device type. | Optional.<br>Data type: object.<br>Optional values: ['cpu', 'npu']. Defaults to 'cpu'. |
| dev_id | Input | Device ID. | Optional.<br>Data type: int.<br>The default value is None.<br>Only takes effect when "dev_type" is configured as "npu". The Device ID specified by "dev_id" has a higher priority than the Device ID configured by environment variables. |
| act_method | Input | Activation quantization method. | Optional.<br>Data type: int. Optional values are as follows, defaults to 1.<br>(1) 1 represents the min-max quantization method for Label-Free scenarios.<br>(2) 2 represents the histogram quantization method for Label-Free scenarios.<br>(3) 3 represents the automatic mixed quantization method for Label-Free scenarios, recommended for LLM large model scenarios.<br>Note: When the lowbit sparse quantization feature is enabled, selecting value 3 is not supported. |
| pr | Input | Quantization selection probability. | Optional.<br>Data type: float. Value range: [0,1].<br>Default value: 1.0, recommended value is 1.0. |
| w_sym | Input | Whether weight quantization is symmetric quantization. | Optional.<br>Data type: bool. Defaults to True.<br>W8A8 and W4A8_DYNAMIC scenarios only support configuration as True.<br>Not applicable to the w4a4 scenario. |
| mm_tensor | Input | Select per_channel quantization or per_tensor quantization. | Optional.<br>Data type: bool. Defaults to True.<br>True: per_tensor quantization.<br>False: per_channel quantization, this quantization method is recommended. |
| w_method | Input | Select the weight quantization strategy. | Optional.<br>Data type: str. Defaults to 'MinMax'. Optional values: 'MinMax', 'GPTQ', 'HQQ', 'NF'.<br>MinMax, HQQ, and NF support Data-Free.<br>GPTQ does not support Data-Free.<br>NF does not support mixed use with per_tensor, per_channel, per_group, lowbit, communication quantization, KV cache quantization, or outlier suppression. |
| co_sparse | Input | Whether to enable the sparse quantization feature. | Optional.<br>Data type: bool. Default value: False, sparse quantization is not enabled.<br>In large model sparse quantization scenarios, the lowbit sparse quantization feature is preferred. After enabling lowbit sparse quantization, the co_sparse parameter automatically becomes invalid. |
| fraction | Input | The proportion of protected outliers during model weight sparse quantization. | Optional.<br>Data type: float. Value range [0.01, 0.1]. The default value is 0.01. |
| nonuniform | Input | Whether to use non-uniform quantization in sparse quantization. | Optional.<br>Data type: bool. The default value is False. |
| is_lowbit | Input | Whether to enable the lowbit quantization feature. | Optional.<br>Data type: bool. Defaults to False, the lowbit quantization feature is not enabled.<br>After setting to True, there are the following two situations:<br>(1) When w_bit=4 and a_bit=8, it is a large model sparse quantization scenario, indicating that the lowbit sparse quantization feature is enabled. In large model sparse quantization scenarios, the lowbit sparse quantization feature is preferred. After enabling lowbit sparse quantization, the co_sparse parameter automatically becomes invalid. `Note: To enable the W4A8_DYNAMIC quantization type, both per_group and per-token need to be enabled simultaneously.`<br>(2) Other scenarios are large model quantization scenarios, which will enable the automatic quantization accuracy tuning feature. The current automatic quantization accuracy tuning framework supports W8A8, W8A16, and W4A8_DYNAMIC quantization.<br>Note: In per_group quantization scenarios, is_lowbit needs to be set to True and open_outlier to False in coordination. |
| do_smooth | Input | Whether to enable the smooth feature.<br>After enabling the do_smooth feature, the activation values are smoothed. | Optional.<br>Data type: bool. Defaults to False, the smooth feature is not enabled. |
| use_sigma | Input | Whether to enable the sigma feature.<br>After enabling the use_sigma feature, outlier protection can be performed based on the characteristics of the normal distribution. | Optional.<br>Data type: bool. Defaults to False, the sigma feature is not enabled. |
| sigma_factor | Input | The value of sigma_factor after enabling the sigma feature, used to limit the protection range of outliers. | Optional.<br>Data type: float.<br>Defaults to 3, value range is [3, 4]. |
| disable_last_linear | Input | Whether to automatically fallback to the last linear layer.<br>When this parameter is currently True, it will automatically fallback the last linear layer. For example, the [lm_head] layer of the LLaMA2-13B model, and the [transformer.output_layer] layer of ChatGLM2-6B. | Optional.<br>Data type: bool.<br>Defaults to True.<br>True: Automatically fallback the last linear layer. False: Will not fallback the last linear layer. |
| use_kvcache_quant | Input | Whether to use the KV cache quantization feature. | Optional.<br>Data type: bool.<br>Defaults to False.<br>True: Use the KV cache quantization feature. False: Do not use the KV cache quantization feature.<br>Note: After setting this parameter to true and configuring the kv_quant parameter in Table 1 Quantization Configuration Table, the KV cache quantization feature can be used. |
| is_dynamic | Input | Whether to use the per-token dynamic quantization feature. | Optional.<br>Data type: bool.<br>Defaults to False.<br>True: Use per-token dynamic quantization. False: Do not use per-token dynamic quantization. |
| open_outlier | Input | Whether to enable weight outlier partitioning. | Optional.<br>Data type: bool.<br>Defaults to True.<br>True: Enable weight outlier partitioning. False: Disable weight outlier partitioning.<br>Note: (1) Only takes effect when lowbit is set to True. (2) In per_group quantization scenarios, is_lowbit needs to be set to True and open_outlier to False in coordination. |
| group_size | Input | The size of the group in per_group quantization. | Optional.<br>Data type: int.<br>The default value is 64, supports configuration as 32, 64, 128, 256.<br>Note: Only applicable to per_group quantization scenarios, requires is_lowbit to be set to True and open_outlier to False in coordination.<br>Not applicable to the w4a4 scenario. |
| percdamp | Input | The matrix positive definite offset coefficient of the GPTQ algorithm, used to ensure the stability of the calculation process. When an error caused by a non-positive definite matrix occurs during GPTQ operation, this parameter can be appropriately increased. | Optional.<br>Data type: float.<br>Value range is [0,1], the default value is 0.01.<br>Note: Only applicable when w_method is the GPTQ algorithm. |
| pdmix | Input | Whether to provide both dynamic quantization parameters and static quantization parameters simultaneously. | Optional.<br>Data type: bool.<br>Defaults to False.<br>True: Generate both dynamic quantization parameters and static quantization parameters simultaneously. False: Only generate a single type of quantization parameter.<br>Note: When set to True, both dynamic and static quantization parameters will be provided simultaneously, making it convenient to choose which parameter type to use based on actual needs during inference. This feature is not supported when is_dynamic=True. |

## Sample 1

Configure all parameters in the QuantConfig initialization according to actual requirements.

```python
from msmodelslim.pytorch.llm_ptq.llm_ptq_tools import Calibrator, QuantConfig
quant_config = QuantConfig(pr=1.0, mm_tensor=False)
```

## Sample 2

After configuring the core parameters (w_bit, a_bit, disable_names, disable_last_linear, dev_type, dev_id) in the QuantConfig initialization, configure the parameters in the table according to different quantization scenarios. Currently supported quantization scenarios include sparse quantization, weight quantization, KV cache quantization, weight-activation quantization, and simulated multi-device quantization. Refer to the table below for specific parameter configurations and call examples.

| Quantization Type | Parameters to Configure | Sample |
| --- | --- | --- |
| weight_quant<br>Parameter initialization for weight quantization, i.e., w8a16.<br>Note: When using a multimodal model, the conv layer will be automatically set to fallback and will not be quantized. | w_method, mm_tensor, w_sym, group_size | quant_config = QuantConfig(w_bit=8,disable_names=disable_names,dev_type='npu',dev_id=0).weight_quant (w_sym=False) |
| weight_activation_quant<br>Parameter initialization for weight and activation quantization, i.e., w8a8. | act_method, pr, is_dynamic | quant_config = QuantConfig(w_bit=8,disable_names=disable_names,dev_type='npu',dev_id=0).weight_activation_quant (act_method=2) |
| sparse_quant<br>Parameter initialization for sparse quantization. | act_method, fraction, nonuniform, is_lowbit, do_smooth, use_sigma, sigma_factor | quant_config = QuantConfig(w_bit=4,disable_names=disable_names,dev_type='npu',dev_id=0).sparse_quant(is_lowbit=True) |
| kv_quant<br>Parameter initialization for KV cache symmetric quantization.<br>Note: After calling this function, use_kvcache_quant will be automatically set to True. | kv_sym is used to enable or disable the KV cache symmetric quantization feature. It is an optional parameter with a data type of bool. The default value is True.<br>True: Use the KV cache symmetric quantization feature. False: Use the KV cache asymmetric quantization feature. | quant_config = QuantConfig(w_bit=8,disable_names=disable_names,dev_type='npu',dev_id=0).kv_quant(kv_sym=True) |
| simulate_tp<br>Parameter initialization for simulated multi-device quantization.<br>Note: By default, simulated multi-device quantization is performed on the linear layers of the communication layer, with each device using different quantization parameters. The simulated multi-device communication quantization feature is then enabled via the enable_communication_quant parameter. | tp_size specifies the number of devices for simulated multi-device quantization. It is a required parameter with a data type of int and a value range of [2, 4, 8, 16].<br>enable_communication_quant is used to enable or disable the simulated multi-device communication quantization feature. It is an optional parameter with a data type of bool and defaults to True.<br>True: Use the simulated multi-device communication quantization scenario. False: Do not use the simulated multi-device communication quantization scenario.<br>enable_per_device_quant is used to configure the method for simulated multi-device communication quantization. It is an optional parameter with a data type of bool. The default value is True.<br>True: Each device uses a different reduce scale. False: Each device uses the same reduce scale. | quant_config=QuantConfig(w_bit=4,disable_names=disable_names,dev_type='npu',dev_id=0).simulate_tp(tp_size=4, enable_communication_quant=True) |
| NF4<br>Parameter initialization for Normal Float 4-bit quantization. | w_method is used to select different weight quantization types. Here, 'NF' quantization is selected.<br>block_size is used to set the number of elements within a block, similar to per_group quantization. A smaller value results in higher quantization precision. It is an optional parameter with a data type of int. The default value is 64, and the value range is [64, 128, 256, 512, 1024, 2048, 4096]. | quant_config = QuantConfig(w_bit=4,a_bit=16,dev_type='npu',dev_id=0).weight_quant(w_method='NF', block_size=64) |
| fa_quant<br>Parameter initialization for FA3 quantization. | fa_amp is used to configure automatic precision fallback, set according to the number of layers to fallback.<br>The data type is int, and the default value is 0. The value range is greater than or equal to 0 and less than or equal to the number of model layers. If it exceeds the number of model layers, the maximum number of model layers will be used as the number of fallback layers. | quant_config=QuantConfig(w_bit=8, a_bit=8, disable_names=disable_names,dev_type='npu',dev_id=0).fa_quant(fa_amp=5) |
