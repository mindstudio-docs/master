## 🆕 如何提交扩展

1. 扩展须**公开可访问**（GitCode/GitHub/GitLab/Gitee 等），含 README 与明确许可证，且与 msServiceProfiler 使用场景相关。
2. **提交方式**：在本仓库 [新建 Issue](https://gitcode.com/Ascend/msserviceprofiler/issues/new)，选择 **「Extension 扩展提交」** 模板，按项填写。
3. **审核**：维护者会在 1～3 个工作日内审核；通过后会将信息**加入本文档**（本目录页）的对应分类表格中。

详细步骤与填写说明见 [社区拓展指南](community_extensions_contributing.md)。

## ❓ 常见问题

**Q: 扩展用起来有问题怎么办？**  
A: 请到该扩展所在仓库提交 Issue，本页仅做索引。

**Q: 如何判断扩展是否可靠？**  
A: 建议查看更新频率、README 与许可证、必要时在小范围环境先试用。

**Q: 可以提交商业或闭源扩展吗？**  
A: 原则上收录开源、可公开访问的扩展；若有免费试用或开源版本且标注清楚许可证，可在提交时说明，由维护者评估。
